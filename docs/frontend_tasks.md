## 📌 현재 미결 과제 (프론트 팀 검토 필요)

### 1. 지도/UI 관련
- **[이슈 #1] 모바일 지도 드래그 불능**: CSS `touch-action` 설정 또는 지도 라이브러리 핸들러 점검 필요.
- **[이슈 #4] 검색 결과 부재**: 검색 시 마커만 노출되고 데이터 목록이나 상세 정보 페이지(Modal/Sidebar)가 뜨지 않는 현상 개선.

### 2. 알림 및 인증
- **[이슈 #7] 카카오 로그인 연동**: 서비스 전환 혹은 콜백 시 라우팅(Deep Link) 오류 확인 필요.
- **[이슈 #8] 토큰 만료 처리**: 로그아웃 후 혹은 토큰 만료 시 적절한 에러 핸들링 및 홈 이동 로직 구현.
- **[이슈 #11] 알림 표시 로직**: SSE 수신 시 Toast UI에 HTML 태그가 그대로 노출되는 현상 수정 (XSS 방지 및 텍스트 렌더링).

### 3. UI 컴포넌트 & 접근성 (2차 QA 신규 이슈)
- **[이슈 P1] 랭킹 "우리 동네 왕" 빈 화면**: 해당 탭 클릭 시 리스트가 빈 배열로 보이는 현상. (백엔드 연동 API 상태도 같이 점검 필요)
- **[이슈 P2] 마이페이지 설정 탭 렌더링 불가**: 프로필 수정 등 설정 탭 진입 시 콘텐츠 렌더링 누락. UI 컴포넌트 렌더링 조건 점검 필요.
- **[이슈 P3] Footer 소셜 링크 미설정**: `href="#"`로 되어 있는 하단 소셜 아이콘 3개 링크에 대한 실제 주소 바인딩.
- **[이슈 P4] `/admin` 우회 시 로딩 UI 노출 문제**: 비인가(일반) 사용자가 어드민 페이지 접근 시, 즉각 리다이렉트되지 않고 "ADMIN GATEWAY LOADING..." 문구가 3초 노출됨. 로딩 화면 없이 즉시 이동(`useEffect` 의존성 확인)되도록 수정.
- **[이슈 P5] 알림 버튼 명확화 (a11y)**: 읽지 않은 알림을 뜻하는 배지(빨간 점) 등에 대한 `aria-label`이 미흡한 부분 보완.

### 4. API 연동 및 기능 개선 (백엔드 완료)
- **[#2] 방문 인증 체결/체류 시간 (완료)**: `POST /api/v1/records/check-in` 호출 시 사용자 실제 진입 시점을 `enteredAt` (Long, timestamp) 필드에 담아 보내면 더욱 정확한 계산이 가능합니다. (미전달 시 서버 수신 시점 기준)
- **[#6] 1:1 문의 삭제 (완료)**: `DELETE /api/v1/support/inquiries/{id}` API가 추가되었습니다. 본인 문의인 경우 삭제 가능합니다.
- **[#5] 리뷰 중복 등록 방지 (완료)**: 백엔드에서 중복 등록 시 `T002 (ALREADY_REVIEWED)` 에러를 반환하므로, 프론트엔드에서 해당 에러 발생 시 안내 메시지를 띄워주세요.
