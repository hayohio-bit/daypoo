# 섹션 6: 이벤트 기반 랭킹 업데이트 (Event Driven)

DayPoo 프로젝트에서는 배변 기록 생성, 리뷰 작성 같은 핵심 서비스 로직과 -> 통계/랭킹 집계 로직을 **서로 다른 트랜잭션과 쓰레드로 느슨하게 결합(Loose Coupling)** 하고 있습니다. 이 연결 고리가 바로 Spring의 `ApplicationEventPublisher` 입니다.

## 1. @TransactionalEventListener (AFTER_COMMIT)의 파워
데이터가 아직 데이터베이스에 커밋(Commit) 되지도 않았는데 레디스의 점수가 오르면 어떻게 될까요? 만약 DB 저장 단계에서 예외가 발생하여 트랜잭션이 롤백 되면, DB의 데이터는 지워지지만 Redis의 점수는 롤백되지 않고 혼자 붕 뜨게 됩니다 (정합성 파괴).

이를 막기 위한 완벽한 해상도가 `@TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)` 입니다.

```java
// PooRecordEventListener.java
@Slf4j
@Component
@RequiredArgsConstructor
public class PooRecordEventListener {

  private final UserRepository userRepository;
  private final RankingService rankingService;
  private final TitleAchievementService titleAchievementService;

  @Async("taskExecutor")
  @TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)
  @Transactional(propagation = Propagation.REQUIRES_NEW)
  public void handlePooRecordCreated(PooRecordCreatedEvent event) {
    // 본 트랜잭션(배변 기록 생성 DB Commit)이 완벽히 성공해야지만 이벤트 리스너가 호출됨.

    User user = userRepository.findByEmail(event.email()).orElseThrow(...);

    // 1. 경험치 추가
    user.addExpAndPoints(event.rewardExp(), event.rewardPoints());
    userRepository.save(user);

    // 2. 외부 서비스(Redis 랭킹 캐시) 반영 호출
    rankingService.updateGlobalRank(user);
    rankingService.updateRegionRank(user, event.regionName());
  }
}
```

## 2. 시퀀스 다이어그램: 비동기 분리 구조
```text
Client -> PooController -> PooService.createRecord()
    |
    |-> (DB) PooRecord INSERT 
    |-> (Event) eventPublisher.publishEvent(new PooRecordCreatedEvent())
    |-> (DB) Transaction 커밋 (응답 반환 속도 극대화)
----------------------------------------------------------------------
    |-> [Redis 비동기 작업] PooRecordEventListener 
        (Main 응답 쓰레드와 무관하게 랭킹 업데이트, Title 체크 등 무거운 연산을 비동기 처리)
```

---

> [!TIP] 랭킹 설계 핵심
> **서비스 비대화 방지 및 속도 최적화**
> 클라이언트 입장에서 가장 중요한 것은 "내 배변 기록이 추가되었다!"는 알림 팝업의 속도입니다. 뒤에서 경험치가 오르고, 레벨이 오르고, 랭킹을 계산하는 로직은 유저에게 바로 리턴해 줄 필요가 없습니다. 메인 비즈니스를 방해하지 않고 **비동기 + 트랜잭션 보장 이벤트** 를 발송하는 기법은 시니어 아키텍처의 꽃입니다.
