# 섹션 1: 랭킹 점수 산정 체계

## 1. 점수 산정 공식 분석
현재 프로젝트의 코드를 분석해보면, 초기 논의되었던 형태의 복잡한 보너스 체계(시간 감쇠, AI 이미지 분석 보너스)보다는 **서비스 핵심 데이터를 명료하게 수치화**하는 방식을 채택하고 있습니다. 별도의 `RankingScoreCalculator` 대신 비즈니스 로직 최상단인 `RankingService` 클래스 내에 점수 산정 규칙이 구현되어 있습니다.

### 글로벌 / 지역 랭킹 점수 로직
- **기본 기록 점수:** 누적 배변 기록 1건당 `1.0점`
- **장소 탐험 보너스:** 서로 다른 화장실에서 기록을 남길 경우 1곳당 `3.0점`의 가중치 부여
- **점수 산출 공식:** `(배변 기록 수) + (구분된 화장실 수 * 3.0)`

### 건강왕 랭킹 점수 로직
- **일일 건강 점수:** `HealthReportSnapshot` 엔티티에 기록된 `healthScore`를 그대로 반영합니다.
- Bristol 배변 스케일과 여러 수치를 종합해 하루 단위의 가장 건강한 유저를 가려냅니다.

---

## 2. Java 코드 예제: RankingService

```java
  public void updateGlobalRank(User user) {
    if (user != null && user.getId() != null) {
      // 1. 점수 요소 조회
      long recordCount = recordRepository.countByUser(user);
      long uniqueToilets = recordRepository.countDistinctToiletsByUser(user);
      
      // 2. 점수 합산 로직
      double score = recordCount + uniqueToilets * 3.0;
      
      // 3. Redis Sorted Set에 저장 (키, 값, 점수)
      redisTemplate.opsForZSet().add("daypoo:rankings:global", user.getId().toString(), score);
    }
  }

  public void updateHealthRank(User user, double healthScore) {
    if (user != null && user.getId() != null) {
        // 건강왕의 경우 전달받은 healthScore를 별도의 가공 없이 저장
      redisTemplate.opsForZSet().add("daypoo:rankings:health", user.getId().toString(), healthScore);
    }
  }
```

## 3. 어뷰징 방지 전략 (공정성 유지)
현재 점수 체계는 어뷰자인 사용자가 동일한 장소에서 연속적으로 기록을 발생시키는 행위를 효과적으로 견제합니다.
- **장소 가중치 부여(x3.0):** 제자리에서 배변 기록 수만 늘리는 것보다, 새로운 화장실을 탐험하여 기록을 남기는 것이 3배의 점수 효율을 가집니다.
- **Rate Limiting:** `AOP` 및 `Redis`를 활용하여 짧은 시간에 비정상적인 기록 호출이 일어나는 것을 방지하고 있습니다.

---

> [!NOTE] 랭킹 설계 핵심
> **심플하지만 직관적인 스코어링**
> 서버 구조에 부담을 주지 않으면서도, "여러 장소에서의 경험"이라는 핵심 기획 의도를 살리기 위해 복잡한 수학 공식 대신 직관적인 상수 곱하기(`* 3.0`) 패턴을 활용합니다. 이는 향후 스케일 아웃 시에도 부하 계산이 매우 간단해지는 장점이 있습니다.
