# 섹션 6: 반응형 레이아웃 및 심층 접근성 가이드

모바일 중심의 서비스인 'DayPoo'는 데스크탑, 태블릿, 모바일까지 부드럽게 레이아웃이 변환(`TailwindCSS` 반응형)되어야 하며, 누구나 차트와 데이터를 읽을 수 있게 **스크린 리더(접근성)** 대응이 필수적입니다.

---

## 1. 뷰포트 맞춤형(Tailwind 4) 차트-리스트 스왑
리팩토링 안에서 랭킹 차트와 리스트가 결합될 때 모바일 화면 크기에 따라 어떻게 공간을 지배할지 분기합니다.

```tsx
<div className="w-full max-w-7xl mx-auto flex flex-col lg:flex-row gap-8 px-4">
  {/* [접근성 유의사항] sm: 이하 화면에서는 차트를 숨기거나 간소화 버전을 보여줌 */}
  <section 
    className="w-full lg:w-1/3 order-2 lg:order-1 hidden md:block" 
    aria-label="랭킹 차트 영역"
  >
    <RankingTrendChart data={chartData} />
  </section>

  {/* 모바일에서는 리스트가 최우선 노출 (100% 강제 너비) */}
  <section 
    className="w-full lg:w-2/3 order-1 lg:order-2 flex-1" 
    aria-label="랭킹 리스트 영역"
  >
    <RankingListContainer users={users} />
  </section>
</div>
```

## 2. 웹 접근성(a11y) 강화
단순한 `div` 파티를 방지하고 시각 장애인용 소프트웨어가 페이지를 읽을 수 있게 시맨틱 태그 구조를 고착화합니다.

```tsx
// RankingList의 경우
<ul role="list" aria-label="전체 사용자 랭킹 리스트">
  {users.map((user) => (
    <li key={user.userId} role="listitem">
       <span className="sr-only">{user.rank}위</span> 
       <span>{user.nick}</span>
    </li>
  ))}
</ul>

// 1위 유저 (왕관 표시)
<span aria-hidden="true">👑</span> {/* 👑 이모지를 음성 리더가 읽지 않게 막음. */}
<span className="sr-only">1위 챔피언 마크</span> {/* 대신 이 텍스트를 읽게 유도함. */}
```

## 3. 색맹/색약 대응 (Color Palette)
Recharts 및 Tailwind 기반 랭킹 개발 시, 적재적소 대비율을 지켜야 합니다.
- **상승(레드)/하락(블루) 지표:** 색상으로만 구분하지 않고, 반드시 **화살표(TrendingUp, TrendingDown)** SVG 아이콘을 조합(`ChangeIcon` 등)하여 모양으로 인지할 수 있게 보장합니다.
- 금/은/동 메달 색깔의 명도 대비(Contrast Ratio)를 WCAG AA 기준인 4.5:1 이상이 되도록 그레이딩합니다.

---

> [!IMPORTANT] Refactoring Point
> **포용적 디자인(Inclusive Design)**
> 접근성 강화는 비단 장애인을 위한 것만이 아닙니다. SEO 시스템 봇(구글 검색 엔진 등)이 페이지의 시맨틱(ul, li, dl 등)과 aria-label을 스캔하면서 `RankingPage`의 정보 품질 점수를 크게 올려줍니다. 컴포넌트 분리 리팩토링 시, 리스트 내의 이모지가 `aria-hidden` 처리되어 있는지 점검하는 것이 매우 중요합니다!
