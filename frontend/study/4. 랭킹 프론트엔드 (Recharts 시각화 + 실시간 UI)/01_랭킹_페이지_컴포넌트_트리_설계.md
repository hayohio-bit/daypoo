# 섹션 1: 랭킹 페이지 컴포넌트 트리 설계

현재 DayPoo의 `RankingPage.tsx`는 **단일 파일 내 1000줄이 넘는 방대한 로직(상태 관리, 애니메이션, 렌더링)**을 모두 가지고 있는 Monolithic(단일 구조) 컴포넌트 형태입니다. 대규모 트래픽과 UI 복잡도 상승을 대비하여 이를 기능별로 잘게 쪼개는 리팩토링 설계도입니다.

## 1. 프론트엔드 모듈화 다이어그램 (Before & After)

### Before: 얽힌 구조 (현재)
```text
RankingPage.tsx (1000+ lines)
 ┣ useRankings (fetching logic)
 ┣ <FlipGlassCard> (Podium item)
 ┣ <Podium> (Top 3)
 ┣ <RankItem> (List item)
 ┣ <MyRankBar> (Fixed bottom bar)
 ┗ <ItemPopup> (Modal state tied to RankingPage)
```

### After: 관심사 분리 아키텍처 (제안)
```text
pages/RankingPage.tsx (컨테이너 역할, RTK Query 구독만 담당)
  ├── RankingHeader (제목, 탭 기능)
  │   └── RankingTabs (total | local | health)
  ├── RankingTopSection 
  │   └── PodiumCard (1, 2, 3위 독립 렌더링)
  ├── RankingChartLayer (신규 Recharts 래퍼)
  │   └── RankTrendChart (변동 그래프)
  ├── RankingListContainer 
  │   └── RankingListItem (가상화 적용된 개별 유저 Row)
  └── MyRankingHighlight (내 순위 고정 바)
```

## 2. 핵심 Props 인터페이스 정의 (TypeScript)

모듈 간 유기적인 데이터 전달을 보장하기 위해 전역 인터페이스를 분리 적용합니다.

```tsx
// types/ranking.d.ts
export interface RankUser {
  userId: number;
  rank: number;
  score: number;
  nick: string;
  level: number;
  title: string;
  titleColor: string;
  titleBg: string;
  avatarUrl: string;       // DiceBear 또는 커스텀 이미지
  effectEmoji?: string;    // 장착된 이펙트 아이템
  change: number;          // 등락율 (양수: 상승, 음수: 하락)
}

// components/ranking/PodiumCard.tsx
export interface PodiumCardProps {
  user: RankUser;
  isFirst?: boolean;
  onSelect: (user: RankUser) => void;
  xOffset?: number; // 3D Repulsion 애니메이션용
}
```

## 3. [Legacy vs Modern] 아키텍처 비교

| 비교 항목 | 현재 (Legacy) | 리팩토링 후 (Modern Component) |
| :--- | :--- | :--- |
| **파일 크기** | `RankingPage.tsx` 단일 1000줄 대배열 | 각 컴포넌트 단위별 100~200줄 내외 통제 |
| **재사용성** | 랭킹 카드를 메인 페이지에서 재사용 불가 | `MyRankingHighlight` 컴포넌트를 언제든 타 화면 이식 가능 |
| **성능 최적화** | 데이터가 변하면 전체 페이지 Re-render | Props가 변경된 개별 컴포넌트만 렌더링 (`React.memo` 도입) |

---

> [!TIP] Refactoring Point
> **컴포넌트 단일 책임 원칙(SRP) 확립**
> 프론트엔드 최적화의 첫 걸음은 상태(State)와 렌더링(View)의 완전한 분리입니다. 무거운 애니메이션(`FlipGlassCard`)과 데이터 상태가 한 페이지 계층에 얽혀 있으면 병목이 발생합니다. 레이아웃과 팝업, 렌더링 컴포넌트를 분할하여 불필요한 메인 스레드 블로킹을 해소하십시오.
