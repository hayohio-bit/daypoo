# 섹션 2: 6개 컨트롤러 통합 API 엔드포인트 설계

**[정합성 검증 완료 사항]**
> 기획 시 추정하셨던 URI와 실제 컨트롤러 소스 코드 간에 약간의 차이가 존재했습니다. (예: `AdminUserController`에 상태 변경이나 특정 유저 기록 조회 API는 존재하지 않으며, 물리적 삭제가 구현됨) 
> 본 문서는 **실제 `backend/src/main/java/com/daypoo/api/controller/` 내부 6개의 Admin 컨트롤러 코드명과 완전 일치**하는 설계도입니다.

## 6대 컨트롤러 API 엔드포인트 매트릭스 백서

### 1. AdminController (대시보드 통계 및 전체 설정 관리)
- 기준 URL: `/api/v1/admin/`

| HTTP Method | API Endpoint | 기능 설명 | 주요 DTO |
| :--- | :--- | :--- | :--- |
| `GET`  | `/stats`         | 대시보드 통계 현황 반환 (회원/상점/구독 지표) | `AdminStatsResponse` |
| `GET`  | `/logs`          | 시스템 내 발생한 중요 이벤트 로그 이력 조회 | `SystemLogResponse` |
| `GET`  | `/settings`      | 전역 시스템 설정 정보 조회 | `SystemSettingsResponse` |
| `PUT`  | `/settings`      | 앱 전역 설정(점검/가입제한) 업데이트 | `SystemSettingsUpdateRequest` |
| `POST` | `/sync-toilets`  | [자동화] 외부 공공 화장실 API -> DB 및 OpenSearch 배치 동기화 | (Map 형태로 결과) |
| `GET`  | `/sync-toilets/status` | 최신 동기화 실행 상황 여부 파악 | `SyncStatusResponse` |
| `POST` | `/rebuild-rankings` | 랭킹 시스템(Redis) 강제 재계산/재구축 | (성공 메시지 string) |
| `POST` | `/generate-test-data` | 관리자 그래프 테스트를 위한 임의 더미 데이터 주입 | (성공 메시지 string) |

### 2. AdminUserController (회원 관리)
- 기준 URL: `/api/v1/admin/users/`

| HTTP Method | API Endpoint | 기능 설명 | 주요 DTO |
| :--- | :--- | :--- | :--- |
| `GET`  | `/`              | 회원 목록 (이메일/닉네임 검색, 구독/롤 필터링 포함) | `Page<AdminUserListResponse>` |
| `GET`  | `/{id}`          | 특정 회원 상세 지표(총 결제, 경험치 등) 확인 | `AdminUserDetailResponse` |
| `PATCH`| `/{id}/role`     | 회원 권한 등업/강등 변경 조치 | `AdminRoleUpdateRequest` |
| `DELETE`|`/{id}`          | 해당 회원의 모든 영속성 데이터를 물리 삭제 (Hard Delete) | - |

### 3. AdminToiletController (화장실 관리)
- 기준 URL: `/api/v1/admin/toilets/`
*(현재 구조에선 단순 CRUD 보단 데이터 플랫폼/인덱싱 동기화에 더 초점이 맞춰져 있음)*

| HTTP Method | API Endpoint | 기능 설명 | 주요 DTO |
| :--- | :--- | :--- | :--- |
| `GET`  | `/count`         | OpenSearch에 인덱싱 된 전체 화장실 개수 반환 | `Long` |
| `GET`  | `/`              | 전체 화장실 목록 페이징 및 키워드 통합 검색 | `Page<AdminToiletListResponse>` |
| `GET`  | `/reindex`       | OpenSearch 데이터 강제 100% 덮어쓰기 재빌드 | (String) |
| `POST` | `/ai-summaries/generate` | (AI 기능) 리뷰 수 5개 이상 화장실에 대해 AI 요약 일괄 도출 자동화 | (Map Count) |

### 4. AdminShopController (상점 관리)
- 기준 URL: `/api/v1/admin/shop/items/`

| HTTP Method | API Endpoint | 기능 설명 | 주요 DTO |
| :--- | :--- | :--- | :--- |
| `GET`  | `/`              | 상점 판매 아이템(타입+검색어) 복합필터 리스트 | `Page<ItemResponse>` |
| `POST` | `/`              | 신규 아바타/테마 아이템 생성 등록 | `AdminItemCreateRequest` |
| `PUT`  | `/{id}`          | 개별 아이템 가격 할인가/이미지/정보 수정 | `AdminItemUpdateRequest` |
| `DELETE`|`/{id}`          | 아이템 데이터 완전 파기 (보유 유저 존재 시 예외 터짐) | - |
| `POST` | `/publish-all`   | 미공개 아이템을 대량으로 한 번에 '판매 중' 토글 공시 | (Count) |
| `PUT`  | `/{id}/toggle-publish` | 아이템 개별 비공개 ↔ 공개 상태 반전 | `ItemResponse` |

### 5. AdminTitleController (칭호 관리)
- 기준 URL: `/api/v1/admin/titles/`

| HTTP Method | API Endpoint | 기능 설명 | 주요 DTO |
| :--- | :--- | :--- | :--- |
| `GET`  | `/`              | 조건부 칭호 리스트업 | `Page<AdminTitleResponse>` |
| `POST` | `/`              | 신규 칭호/목표(Threshold) 생성 | `AdminTitleCreateRequest` |
| `PUT`  | `/{id}`          | 기존 칭호 수정 | `AdminTitleUpdateRequest` |
| `DELETE`|`/{id}`          | 사용 중이지 않은 칭호 파기 | - |

### 6. AdminInquiryController (문의 CS 플랫폼)
- 기준 URL: `/api/v1/admin/inquiries/`

| HTTP Method | API Endpoint | 기능 설명 | 주요 DTO |
| :--- | :--- | :--- | :--- |
| `GET`  | `/`              | PENDING 상태 등 질의별 문의 목록 | `Page<AdminInquiryListResponse>` |
| `GET`  | `/{id}`          | 민원 상세 내용 열람 | `AdminInquiryDetailResponse` |
| `POST` | `/{id}/answer`   | 1:1 민원에 대해 관리자가 답변 달고 "완료" 변경 | `AdminInquiryAnswerRequest` |
| `POST` | `/generate-test-data`| 개발망 체크상 대기 중인 가짜 CS 인입 테스트 트래픽 발생 | (String) |

---

> [!WARNING] 보안 체크리스트
> 화장실 정보 동기화(`sync-toilets`)나 인덱싱(`reindex`) 작업은 백그라운드 스레드 및 메모리를 엄청나게 잡아먹는 Heavy Query가 수반됩니다. 악의적인 공격자가 해당 관리자 세션을 탈취해 API를 연타하면 서버가 죽어버립니다(DDoS). 
> DayPoo 백엔드는 이를 방어하기 위해 현재 진행 중인 상태(`RUNNING`)를 점검해 무시하거나, 스레드풀을 격리하는 가상 스레드 동기화 비동기 처리 전략을 적절히 사용해야 합니다.
