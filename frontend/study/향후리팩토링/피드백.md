# DayPoo 프로젝트 심층 분석 기반 개선 및 피드백 우선순위

전체 아키텍처 스터디(9개 도메인, `study_Ex` 제외)를 통해 도출된, 프론트/백엔드/인프라 상 가장 시급하게 개선·반영되어야 할 기술 부채 및 피드백 순위입니다.

---

### **1순위: 관리자 `AdminPage.tsx` 모놀리틱 컴포넌트 분할 및 라우터 분리**
> **이유:** 단일 파일이 4,187라인에 달해 프론트엔드 렌더링 병목 및 협업 마비를 유발하므로, 즉각적인 `React Router` 중첩 분리와 코드 스플리팅이 필요합니다.

### **2순위: 관리자 대시보드 통계 로직(`getAdminStats`) 메모리 OOM 방지**
> **이유:** `UserRepository.findAll().stream()` 등 애플리케이션 메모리 위에서 통계를 집계하고 있어 사용자가 늘면 서버가 즉각 다운되므로, JPQL 기반 Native DB 집계 함수로 교체해야 합니다.

### **3순위: AI 리포트/이미지 분석 동기화 통신(Timeout) 병목 해소**
> **이유:** Python AI 서버 통신 지연(10초 이상) 발생 시 화면이 쪼개지며 접속이 끊기므로, 이를 백그라운드 큐 비동기 처리 + 이벤트 알림(`SSE`) 방식으로 전환해야 합니다.

### **4순위: 프론트엔드 전역 상태 관리(`RTK Query`) 및 유효성(`Zod`) 도입**
> **이유:** 기획과 달리 현재 전반적인 API 페칭이 순수 `Axios`와 캐싱 없는 `useEffect` 위주로 하드코딩 되어 있어 브라우저 리소스 낭비가 심합니다.

### **5순위: 랭킹 업데이트 이벤트(`RankingEventListener`) 로직의 알림 수신 자동화**
> **이유:** 점수 누적에는 성공하나 변동된 '순위 상승/하락' 여부를 비교 및 추적해 SSE 알림 `Notification` 레코드로 자동 생성해주는 E2E 파이프라인이 미구현 상태입니다.

### **6순위: 관리자 칭호 수동 제어(부여/회수) 엔드포인트 신설**
> **이유:** 랭킹 이벤트에 의한 '자동 칭호 획득'만 구현되었을 뿐, 기획서 상의 관리자 타겟팅 커스텀 칭호 제어 REST API(`/grant`)가 백엔드에 누락되었습니다.

### **7순위: 대규모 봇 시뮬레이션을 대비한 Redis 파이프라이닝 및 AOP 제한 예외**
> **이유:** `RateLimitAspect`에 의해 더미 테스트 봇 트래픽이 100% 차단당하고, `ZINCRBY` 단일 스레드 병목이 발생해 정상적인 로드 밸런싱 모니터링 테스트가 불가능합니다.
