# 05. 에러 핸들링 및 보안

> 에러 처리 3계층 + JWT 저장 전략 + XSS/CSRF 방어 + CORS 설정 정리

## 에러 처리 3계층

```text
사용자 액션
    │
    ▼
① apiClient.request()          ← 네트워크 레이어
    ├── AbortError              → ApiError { code: 'TIMEOUT',       status: 408 }
    ├── TypeError(fetch fail)   → ApiError { code: 'NETWORK_ERROR', status: 0   }
    ├── 401                     → tryRefreshToken() → 성공 시 재시도 / 실패 시 throw
    └── 4xx / 5xx               → ApiError { code: 백엔드ErrorCode, status, message }
    │
    ▼
② 컴포넌트 catch 블록           ← 컴포넌트 레이어
    ├── 0   NETWORK_ERROR       → "네트워크 연결을 확인해 주세요" 토스트
    ├── 401 (refresh 실패)      → window.location.href = '/login'
    ├── 403 FORBIDDEN           → "접근 권한이 없습니다" 토스트
    ├── 404 NOT_FOUND           → "데이터를 찾을 수 없습니다" 토스트
    ├── 408 TIMEOUT             → "요청 시간이 초과되었습니다" 토스트
    ├── 429 TOO_MANY_REQUESTS   → "잠시 후 다시 시도해 주세요" 토스트
    └── 500+                    → window.location.href = '/server-error'
    │
    ▼
③ ErrorBoundary                ← 앱 레이어 (최후 보루)
    └── 렌더링 중 JS 예외 포착 → 에러 UI (새로고침 + 홈 버튼)
```

## ApiError 구조

```typescript
// apiClient.ts에서 모든 에러를 이 형태로 통일
export interface ApiError extends Error {
  code: string;   // "TIMEOUT" | "NETWORK_ERROR" | "AUTHENTICATION_REQUIRED" | 백엔드 ErrorCode
  status: number; // 0(네트워크) | 408(타임아웃) | 401~500(HTTP)
}

// 컴포넌트에서 사용 패턴
try {
  const data = await someServiceFunction();
} catch (err) {
  const error = err as ApiError;
  switch (error.status) {
    case 0:   return showToast('error', '네트워크 연결을 확인해 주세요.');
    case 401: return (window.location.href = '/login'); // refresh도 실패한 상태
    case 403: return showToast('error', '접근 권한이 없습니다.');
    case 404: return showToast('error', '데이터를 찾을 수 없습니다.');
    case 429: return showToast('error', '잠시 후 다시 시도해 주세요.');
    default:  error.status >= 500
                ? (window.location.href = '/server-error')
                : showToast('error', error.message || '알 수 없는 오류가 발생했습니다.');
  }
}
```

## ErrorBoundary 주의사항

```tsx
// components/ErrorBoundary.tsx — Class 컴포넌트 필수 (React 19도 동일)
// ✅ 포착 가능: 렌더링 중 throw된 에러
// ❌ 포착 불가: 이벤트 핸들러(onClick 등), async/await, setTimeout

// ❌ 포착 불가 → 전역 리스너로 별도 처리 (선택적)
window.addEventListener('unhandledrejection', (event) => {
  console.error('Unhandled rejection:', event.reason);
});

// main.tsx — 앱 최상단에 배치
<ErrorBoundary>
  <App />
</ErrorBoundary>
```

| --- | --- | --- |

## CORS 설정 (Spring Security)

```java
// SecurityConfig.java — 환경별 Origin 환경변수로 관리
config.setAllowedOrigins(allowedOrigins);  // app.cors.allowed-origins (application.yml)
config.setAllowedMethods(List.of("GET", "POST", "PUT", "DELETE", "PATCH", "OPTIONS"));
config.setAllowedHeaders(List.of("Authorization", "Content-Type", "Accept", "X-Correlation-Id"));
config.setAllowCredentials(true); // OAuth2 + JWT 쿠키 전송 필요
config.setMaxAge(3600L);          // Preflight 캐시 1시간 (OPTIONS 요청 횟수 감소)
```

```yaml
# application-local.yml
app.cors.allowed-origins:
  - "http://localhost:5173"  # Vite dev server

# application-prod.yml
app.cors.allowed-origins:
  - "https://daypoo.8o2.site"  # CloudFront 도메인
```

## 환경별 배포 흐름

```text
로컬 개발
  브라우저:5173 → Vite proxy → localhost:8080 (Spring Boot)
  /api, /oauth2, /login/oauth2 모두 프록시 처리

프로덕션 (CloudFront + S3 + EC2)
  브라우저 → CloudFront
    /api/*          → EC2 (Spring Boot)
    /oauth2/*       → EC2 (OAuth2 인증)
    /login/oauth2/* → EC2 (OAuth2 콜백)
    나머지           → S3 (React SPA) → 404/403 → index.html (SPA 라우팅)
```
