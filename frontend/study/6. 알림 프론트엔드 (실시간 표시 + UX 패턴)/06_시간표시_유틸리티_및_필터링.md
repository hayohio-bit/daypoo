# 섹션 6: 시간 표시 유틸리티 & 필터링 렌더링

**[정합성 검증 완료 사항]**
> 사용자가 제시한 `timeAgo.ts` 모듈은 분리되어 있지 않고, 현재 `NotificationPanel.tsx` 내부 헬퍼 함수로 포함되어 동작 중입니다. 필터 탭도 `all`과 `unread` 2가지의 명확하고 단순화된 뎁스(Depth)로 구현되어 있습니다.

## 1. 시간 표시 포맷터 (getTimeAgo)

SNS 스타일의 친화적인 상대적 시간 표현 규격입니다. 현재 인라인으로 작성되어 있지만, 향후 복잡도를 고려해 유틸 폴더로 빼는 리팩토링 스펙을 고려해 볼 만합니다.

```tsx
// NotificationPanel.tsx 내장 구현부
const getTimeAgo = (dateString: string) => {
  const now = new Date();
  const date = new Date(dateString);
  const diffMs = now.getTime() - date.getTime();
  
  // 단위 구간 정의
  const diffMins = Math.floor(diffMs / 60000);
  const diffHours = Math.floor(diffMs / 3600000);
  const diffDays = Math.floor(diffMs / 86400000);

  // 조건 분기
  if (diffMins < 1) return '방금 전';
  if (diffMins < 60) return `${diffMins}분 전`;
  if (diffHours < 24) return `${diffHours}시간 전`;
  if (diffDays < 7) return `${diffDays}일 전`;
  
  // 1주일 경과 시 절대 날짜 표기 전환
  return date.toLocaleDateString('ko-KR', { month: 'short', day: 'numeric' });
};
```

## 2. 알림 탭(Tabs) 액션과 필터 메모이제이션

복잡한 4구간 필터가 아닌 **전체(all)** / **미읽음(unread)** 이분화 논리를 갖춰 사용자의 시각적 스트레스를 줄이고 있습니다.

```tsx
type FilterType = 'all' | 'unread';
const [filter, setFilter] = useState<FilterType>('all');

// useMemo를 통한 필터링 로직 (notifications 원본 데이터 배열 보존 원칙)
const filteredNotifications = useMemo(() => {
  return filter === 'unread'
    ? notifications.filter(n => !n.isRead) // 읽지 않은 항목들만 필터
    : notifications; // 전체 리스트 반환
}, [notifications, filter]);
```

### 3. 읽음 상태 표시 시각화

필터에 따라 걸러진 항목은 Tailwind의 `isRead` 삼항 연산자를 바탕으로 배경색(Background) 투명도와 테두리로 안읽은 티를 팍팍 내줍니다.

```tsx
<motion.div
  // 읽으면 흐려지고 배경이 투명해지며 융화됨. 안 읽으면 새하얗고 그림자가 짐.
  className={`relative group p-4 rounded-2xl transition-all ${
    n.isRead
      ? 'bg-white/50 backdrop-blur-sm'
      : 'bg-white backdrop-blur-md shadow-md hover:shadow-lg'
  }`}
  style={{
    border: n.isRead ? '1px solid rgba(0,0,0,0.05)' : '1px solid rgba(45,106,79,0.1)',
  }}
>
  {/* 미읽음 상태일 때 좌측 상단 푸른색 인디케이터 구슬 렌더링 */}
  {!n.isRead && (
    <div className="absolute top-4 right-4 w-2 h-2 rounded-full bg-gradient-to-br from-emerald-400 to-teal-500" />
  )}
  ...
</motion.div>
```

> [!TIP] UX 플로우 전체 리뷰
> **백그라운드에서 드롭다운까지 심리스 워크플로우** <br>
> 1. `NotificationSubscriber`가 SSE `message` 트리거 수신
> 2. `NotificationContext`가 `showToast()` 발동 ➡️ `NotificationToast` 슬라이드 등장
> 3. `fetchNotifications()` 동시 호출로 Context 내부 카운팅 +1 증가
> 4. `Navbar` 컴포넌트 내 Bell 아이콘에 빨간색 미읽음 뱃지 업데이트!
> 5. 유저가 Bell을 클릭 ➡️ `NotificationPanel(isOpen=true)` 애니메이션 전환
> 6. 안읽은 알림 클릭 ➡️ API `/read` 및 로컬 상태 전환 + `redirectUrl`로 컴포넌트 즉각 이동!
