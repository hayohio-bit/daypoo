# 섹션 1: 알림 컴포넌트 트리 설계 및 분석

DayPoo 프로젝트의 실제 프론트엔드 코드베이스 구조를 파악하고 검증한 결과를 바탕으로 한 알림 시스템 컴포넌트 트리 다이어그램입니다.

## 1. 컴포넌트 트리 다이어그램 (텍스트 구조화)

실제 코드에서 알림 시스템은 전역 래퍼(`NotificationProvider`)와 최상단 구독자(`NotificationSubscriber`) 구성을 통해 단일 진실 공급원(Single Source of Truth)을 확보하고 있습니다.

```text
App.tsx
 ├── NotificationProvider (Context API - 전역 상태 및 로직)
 │    ├── <NotificationSubscriber /> (백그라운드 SSE 구독 리스너)
 │    ├── <Navbar>
 │    │    └── NotificationBell (헤더 종 모양 아이콘 + 읽지 않은 뱃지 클릭 트리거)
 │    ├── <NotificationPanel> (모바일/데스크탑 대응 패널 드롭다운 겸 전용 목록 역할)
 │    │    ├── FilterTabs (전체 / 미읽음)
 │    │    ├── EmptyState (알림이 없을 때 표시)
 │    │    └── NotificationListItem (반복 렌더링 카드)
 │    └─ AnimatePresence (Fixed Bottom 컨테이너)
 │         └── <NotificationToast> (신규 알림 실시간 팝업)
```

## 2. 주요 컴포넌트 프롭스(Props) 및 타입 인터페이스 정의

알림 도메인에서 사용되는 핵심 인터페이스는 `NotificationContext.tsx` 시스템에 정의되어 있습니다.

```tsx
// 2-1. 단일 알림 객체 타입
export interface Notification {
  id: number;
  type: 'INFO' | 'ACHIEVEMENT' | 'INQUIRY_REPLY' | 'SYSTEM';
  title: string;
  content: string;
  isRead: boolean;
  createdAt: string; // ISO 8601 포맷
  redirectUrl?: string;
}

// 2-2. NotificationToast Props
export type ToastType = 'info' | 'achievement' | 'message' | 'star' | 'error';
export interface NotificationToastProps {
  type: ToastType;
  title: string;
  message: string;
  icon?: string;
  onClose: () => void; // 유저가 X 버튼을 눌렀을 때 사라짐
}

// 2-3. NotificationPanel Props
export interface NotificationPanelProps {
  isOpen: boolean;       // Navbar 등에서 벨을 클릭했을 때 true/false 제어
  onClose: () => void;
}
```

## 3. EmptyState (알림이 없을 때)

드롭다운 또는 전체 패널을 열었으나 보여줄 데이터가 없을 때, 사용자에게 시각적 안정감을 주도록 Framer Motion을 결합해 디자인되어 있습니다.

```tsx
// 실제 NotificationPanel.tsx에 구현된 Empty State JSX 조각
<motion.div
  initial={{ opacity: 0, y: 20 }}
  animate={{ opacity: 1, y: 0 }}
  className="h-full flex flex-col items-center justify-center text-center px-6"
>
  <motion.div
    animate={{ scale: [1, 1.05, 1], rotate: [0, 5, -5, 0] }}
    transition={{ duration: 3, repeat: Infinity, ease: "easeInOut" }}
    className="w-24 h-24 rounded-3xl bg-gradient-to-br from-emerald-50 to-teal-50 flex items-center justify-center mb-6"
  >
    <Sparkles size={40} className="text-emerald-300" />
  </motion.div>
  <h3 className="font-black text-lg text-[#1a2b22] mb-2">새로운 알림이 없어요</h3>
  <p className="text-sm text-[#7a9e8a] leading-relaxed">
    오늘 변은 어떠셨나요? <br /> 새로운 알림이 도착하면 알려드릴게요!
  </p>
</motion.div>
```

---

> [!TIP] UX 설계 팁
> **통합 패널 구조 전략** <br>
> 별도의 라우팅된 전체 페이지(`NotificationsPage`)를 두지 않고, 데스크탑에서는 오프캔버스 드롭다운(Off-Canvas Dropdown)으로 작동하고 모바일에서는 전체 화면을 채우는 `NotificationPanel` 1개로 모으는 전략은 상태 관리가 꼬이지 않고 심리스(Seamless)한 UX 전환을 제공합니다.
