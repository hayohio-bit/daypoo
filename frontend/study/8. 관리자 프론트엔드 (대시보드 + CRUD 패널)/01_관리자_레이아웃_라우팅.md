# 섹션 1: 관리자 레이아웃 & 싱글 페이지 상태 라우팅

**[정합성 검증 완료 사항]**
> `App.tsx`와 실제 `AdminPage.tsx` 소스 코드를 분석한 결과, 기획상의 React Router 중첩 라우트(`<Route path="users">...`) 구조는 사용되지 않고 있습니다. 
> 백오피스 전체 화면이 거대한 하나의 **`AdminPage.tsx` 싱글 파일 모놀리틱 아키텍처**로 구성되어 있으며, URL 이동 없이 `useState<AdminTab>` 상태를 컨트롤하여 `switch-case` 혹은 조건부 렌더링으로 탭 간 화면 전환을 수행합니다.

## 1. 현재의 모놀리틱 탭 기반 라우팅 로직

하나의 페이지에서 레이아웃(사이드바, 헤더)과 각 패널 콘텐츠들을 전부 응집시켜 보여주는 방식입니다. 라우팅 오버헤드 없이 즉각적인 전환이 장점입니다.

```tsx
// frontend/src/pages/AdminPage.tsx 구조 파악

type AdminTab = 'dashboard' | 'users' | 'toilets' | 'cs' | 'store' | 'titles' | 'system' | 'add-item' | 'logs';

export function AdminPage() {
  const [activeTab, setActiveTab] = useState<AdminTab>('dashboard');

  return (
    <div className="flex h-screen bg-gray-50 overflow-hidden">
      {/* 고정 사이드바 (AdminSidebar) 기능 내장 */}
      <aside className="w-64 bg-white border-r">
          <nav>
            <button onClick={() => setActiveTab('dashboard')} className={activeTab === 'dashboard' ? 'active' : ''}>
               대시보드
            </button>
            <button onClick={() => setActiveTab('users')} className={activeTab === 'users' ? 'active' : ''}>
               회원 관리
            </button>
            {/* ... */}
          </nav>
      </aside>

      {/* 메인 콘텐츠 영역 (Outlet 대신 렌더링 함수 사용) */}
      <main className="flex-1 overflow-y-auto">
        <header>...</header>
        <div className="p-8">
          {activeTab === 'dashboard' && <DashboardView setActiveTab={setActiveTab} />}
          {activeTab === 'users' && <UsersTab />}
          {activeTab === 'toilets' && <ToiletsTab />}
          {activeTab === 'cs' && <CsTab />}
          {/* ... */}
        </div>
      </main>
    </div>
  );
}
```

## 2. 향후 React Router 중첩 라우팅 도입 방향

모놀리틱 파일 크기가 4,000줄을 넘어 리팩토링이 필요할 때, 기획안과 같은 아래 형태의 코드로 분리할 것을 권장합니다.

```tsx
// [도입 예시] 코드 스플리팅 적용을 위한 App.tsx 라우터 리팩토링 목표
<Route element={<AdminRoute />}>
  <Route path="/admin" element={<AdminLayout />}>
    <Route index element={<DashboardPage />} />
    <Route path="users" element={<UsersPage />} />
    <Route path="toilets" element={<ToiletsPage />} />
    <Route path="shop" element={<ShopPage />} />
  </Route>
</Route>
```

---

> [!TIP] 관리자 UI 패턴
> **Monolithic Component 분리 전략** <br>
> 파일이 너무 커지면 프론트엔드 렌더링 성능 및 팀 협업에 병목이 발생합니다. 먼저 제일 뚱뚱한 `DashboardView`와 `UsersTab`의 코드를 뜯어내어 `components/admin/tabs/` 폴더 등으로 옮기는 작업(코드 스플리팅 파편화)을 1순위로 진행하는 것을 추천합니다.
